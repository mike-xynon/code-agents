# Operating Rules

1. Always update `report.md` when you complete a task or hit a blocker.
2. Before modifying code, update `design.md` with your approach.
3. Reference code changes as: `<repo>:<file>:<line>` (e.g., `registry:src/Hin/Transfer.cs:142`)
4. If blocked, message parent via inbox.
5. Do not push to main branches without parent approval.
6. Check inbox periodically for control messages.
7. When spawning children, provide clear init.md and governing.md.
